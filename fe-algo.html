9-1科目Bの戦い方 — 100分20問の時間配分

🎯 ゴール：科目Bの出題構成と時間配分の作戦を知り、本番で無駄に慌てないようにする。

科目Bは100分で20問。単純計算では1問あたり平均5分ですが、問題ごとの難易度差が大きいので「必ず5分」ではなく目安として使います。まずは全体の内訳を押さえましょう。

種別問数配点イメージおすすめの順番
情報セキュリティ（シナリオ型）4問2割① 先に片づける
アルゴリズム（擬似言語）16問8割② 時間を多く残す

セキュリティのシナリオ問題は読解勝負でトレース不要なぶん時間を予測しやすいので、先に解いて残り時間をアルゴリズムに回すのが定番の作戦です。アルゴリズムは1問の中でもトレースに時間がかかる設問とすぐ分かる設問が混ざっているので、詰まったら印だけ付けて次に進み、最後に見直す方が全体の得点は伸びます。なお設問形式は、値を答えるトレース型よりも『プログラムの空欄に入る字句の組合せを選ぶ』穴埋め型が最多です（9-6の例題でこの形式を体験できます）。

⚡ ポイント
  • 合格ラインは600/1000点。満点を狙う試験ではないので、分からない1問に時間を溶かさない。
  • 先にセキュリティ4問→残り時間をアルゴリズムに使う、が合格体験記で共通する戦略。
  • 全問にいったんマークしてから見直す。飛ばした問題を忘れて時間切れにしない。

9-2宣言・代入・演算 — 擬似言語の基本文法①

🎯 ゴール：擬似言語の記法（宣言・代入・呼出し・演算子の優先順位）をIPA公式の表どおりに読めるようになる。

科目Bのプログラムは「擬似言語」という、IPAが公式に決めた共通の書き方で出題されます。実在するプログラミング言語（Python・Javaなど）ではないので、まずこの資料の記法だけを覚えれば十分です。下の表がIPA公式仕様（Ver.5.0）の記法一覧の一部です。

この表の書き方を実際に使うとこうなります。

○ goukei(整数型: n)   /* goukeiという関数の宣言。引数nは整数型 */
  整数型: i, s          /* i と s を 整数型で変数宣言 */
  s ← 0                /* 代入：sに0を入れる */
  goukei(3)             /* 呼出し：goukeiにひきすう3を渡して実行 */

演算子には優先順位があり、上から先に計算されます（高→低）。この順番は本文中でも一字一句変えずに使うので、そのまま覚えてください。

( ) と .（メンバ参照）　＞　単項 not, +, −　＞　mod, ×, ÷　＞　+, −　＞　関係(≠ ≦ ≧ < = >)　＞　and　＞　or

modは割り算の余りを求める演算子です（例：7 mod 2 = 1）。「掛け算・割り算・modが先、足し算・引き算は後、比較はもっと後、and/orは一番後」という順番だけ押さえれば式は読めます。

⚡ その他の規約
  • 論理型の定数は true / false。
  • 配列は [ ] で要素番号を指定。1始まりかどうかは必ず問題文で明示される。二次元配列は「行,列」の順。{ }は配列リテラル（二次元は内側の{}が1行分）。
  • 「未定義」「未定義の値」＝値が入っていない状態を表す公式用語。連結リストの終端などでよく出てくる。

9-3制御構造 — 擬似言語の基本文法②

🎯 ゴール：if/while/do-while/forの4つの構文と、それぞれの実行順の違いを読み解けるようになる。

選択と繰返しの構文はこの4種類です。「上から評価して最初に真になった所だけ実行」「前判定か後判定か」の違いがひっかけになりやすいポイントです。

if (条件式1)
  …
elseif (条件式2)
  …
else
  …
endif

if文は上から順に条件式を評価し、最初に真になった処理だけを実行します（elseifは複数書けて、省略もできます）。

while (i ≦ n)
  …
  i ← i + 1
endwhile

do
  …
  i ← i + 1
while (i ≦ n)

for (i を 1 から n まで 1 ずつ増やす)
  …
endfor
⚡ ポイント
  • whileは条件を先に確認する「前判定」。最初から条件が偽なら本体は1回も実行されない。
  • do～whileは本体を先に実行してから条件を確認する「後判定」。条件が最初から偽でも必ず1回は実行される。ここが最頻出のひっかけ。
  • forは初期値・繰返し条件・更新をひとまとめに書ける繰返し。endif/endwhile/endforで必ず閉じる。

9-4トレース表の書き方 — この章の心臓部

🎯 ゴール：擬似言語のプログラムを1行ずつ手を動かして追い、変数の値の変化を表に書き出せるようになる。

科目Bの正体は「暗記」ではなく「トレース（プログラムを1行ずつ実際に実行した気持ちで追うこと）」です。手順はいつも同じ4ステップです。

  1. ①ゴール確認：何を答える問題か（戻り値？出力？空欄の穴埋め？）を先に確認する。
  2. ②初期値を書く：変数が最初に持っている値を表の1行目に書く。
  3. ③列を決める：値が変わる変数だけを表の列にする（変わらない変数は書かなくてよい）。
  4. ④1行ずつ実行：コードを上から1行ずつ実行し、値が変わるたびに表に1行足す。
✏️ 例題（本試験風） 次の擬似言語で書かれた手続を kazoeru(6) として呼び出したとき、戻り値はいくつになるか。 
○ kazoeru(整数型: n)
  整数型: i, kosuu
  kosuu ← 0
  i ← 1
  while (i ≦ n)
    if (i mod 3 = 0)
      kosuu ← kosuu + 1
    endif
    i ← i + 1
  endwhile
  return kosuu
答えを見る

トレース表で1周ずつ確認します（i mod 3 = 0、つまり3の倍数のときだけkosuuが増えます）。

ii mod 3kosuu
11（≠0）0
22（≠0）0
301
41（≠0）1
52（≠0）1
602

i=7になった時点でi≦6が偽になりループ終了。戻り値は2（1〜6のうち3の倍数は3と6の2個）。

⚡ ポイント
  • 「値が変わる変数だけ」を列にする。ループカウンタや一時変数まで全部書くと逆に混乱する。
  • 行は「値が変わったとき」だけ増やす。1行ずつ機械的にコードをなぞるのがコツ。
  • 練習では正確さ優先で1問5〜7分かけてよい。省略できる列を見極めながら、本番の平均5分ペースに近づけていく。

9-5配列のトレース — 添字ズレに要注意

🎯 ゴール：配列の要素番号（添字）の数え方を確認し、添字ズレ（オフバイワン）のミスを防げるようになる。

配列は要素番号（添字）が1始まりか0始まりかで動きが変わります。基本情報の擬似言語では1始まりで出題されることが多いですが、必ず問題文の宣言を確認してください。この「思い込みによる添字ズレ（オフバイワン）」が科目Bの最多ミスと言われています。

二次元配列は「行,列」の順で書きます（例：a[2,3]は2行目3列目）。表計算のイメージに近いですが、順番が「行が先」であることを忘れないようにしましょう。

⚡ ポイント
  • 問題文に「配列の要素番号は1から始まる」のような一文が必ずある。まずそこを探す。
  • ループの終了条件（i ≦ n か i < n か）と添字の始まりが対応しているか確認する。
  • 2つの値を入れ替える（スワップ）ときは、一時変数（tmp）に一旦逃がすのが定番の型。

9-6関数と引数 — 値の受け渡しと戻り値

🎯 ゴール：関数を呼び出したときに、引数がどう渡り、戻り値がどう返ってくるかを図でイメージできる。

関数（手続）を呼び出すと、処理はいったんその関数へ「飛んで」、関数の中身を実行し終えたら、呼び出した場所の続きに「戻ってくる」動きをします。

○ nibai(整数型: x)
  return x × 2

/* 呼出し側 */
kekka ← nibai(5)

nibai(5)が呼ばれると、関数の中ではxに5が入り、x × 2 = 10を戻り値として返します。呼び出し側のkekkaにはその10が代入されます。

✏️ 例題（本試験風・穴埋め型） 実際の科目Bで最も多い設問形式は『空欄穴埋め（組合せ選択）』です。ここで形式に慣れておきましょう。次の関数saidaiは、要素数n（n≧1）の整数型の配列aの最大値を返す（要素番号は1始まり）。［　a　］・［　b　］に入る字句の適切な組合せはどれか。 
○ saidai(整数型の配列: a, 整数型: n)
  整数型: i, m
  m ← ［ a ］
  for (i を 2 から n まで 1 ずつ増やす)
    if ( ［ b ］ )
      m ← a[i]
    endif
  endfor
  return m
ab
アa[1]a[i] ＞ m
イa[1]a[i] ＜ m
ウ0a[i] ＞ m
エ0a[i] ＜ m
答えを見る

答えはアです。

［b］は「暫定の最大値mより大きいときだけmを更新する」という意味なので、a[i] ＞ mが入ります。

［a］はmの初期値です。0で初期化してしまうと（ウ・エ）、配列の要素がすべて負の数（例：{−3, −1, −5}）だったとき、どの要素もmの0を超えられず、誤って0を返してしまいます。配列の1番目の値a[1]で初期化しておけば、要素数が1個のときも含めて必ず正しい値からスタートできます。

解き方のコツ：まず空欄1つ1つの役割を日本語で言ってみる（「mの初期値」「更新条件」）→選択肢どうしの差分だけを比較する→極端な入力（全部負の配列・要素数1個）を当てはめて誤りの選択肢を消去する、の3ステップです。

⚡ ポイント
  • 引数は「関数に渡す入力」、戻り値（return）は「関数からの出力」。向きを逆にしない。
  • 関数呼び出し中は、呼び出し元の実行がそこで止まり、関数の処理が終わるまで待つ。
  • この「呼んで→戻る」動きが9-7の再帰の土台になる。

9-7再帰の追い方 — マトリョーシカ式の呼び出し

🎯 ゴール：自分自身を呼び出す「再帰」の動きを、呼び出しの積み重ね（コールスタック）で追えるようになる。

再帰とは、関数が自分自身を呼び出すことです。マトリョーシカ人形を開けていくように、どんどん小さい問題を呼び出していき、一番小さい人形（終了条件）にたどり着いたら、そこから答えを組み立てながら外側へ戻っていきます。終了条件を最初に確認するのがコツです。

○ kaijo(整数型: n)
  if (n = 0)
    return 1
  else
    return n × kaijo(n - 1)
  endif

下から順に見ると分かりやすいです。kaijo(3)はまずkaijo(2)を呼び、kaijo(2)はkaijo(1)を呼び、kaijo(1)はkaijo(0)を呼びます。kaijo(0)は終了条件（n=0）なので1を返し、そこから下に向かって kaijo(1)=1×1=1、kaijo(2)=2×1=2、kaijo(3)=3×2=6 と答えを組み立てながら戻っていきます（3の階乗=6）。

✏️ 例題（本試験風） hyoji(x)はxの値を画面に出力する手続きとする。次の手続きsaiki(3)を呼び出したとき、画面に出力される数字の順番はどうなるか。 
/* hyoji(x)はxの値を画面に出力する手続き */
○ saiki(整数型: n)
  if (n > 0)
    hyoji(n)
    saiki(n - 1)
    hyoji(n)
  endif
答えを見る

呼び出しを積む方向と、戻る方向の2回、hyojiが実行されることに注意してトレースします。

呼び出し1回目のhyoji（積む方向）再帰呼出し2回目のhyoji（戻る方向）
saiki(3)3を出力saiki(2)を呼ぶ3を出力
saiki(2)2を出力saiki(1)を呼ぶ2を出力
saiki(1)1を出力saiki(0)を呼ぶ1を出力
saiki(0)n=0なのでif文が偽→何もせず戻るだけ（出力なし）

積む方向（3→2→1）、底で何もせず折り返し、戻る方向（1→2→3）の順で出力されるので、出力順は 3 2 1 1 2 3 です。

⚡ ポイント
  • 終了条件（base case）を必ず最初に確認する。書かないと呼び出しが無限に続いてしまう。
  • 紙に呼び出しを縦に積み上げて書くと、頭の中だけで追うより圧倒的に混乱しない。
  • 「呼ぶ前の処理」と「呼んだ後（戻ってきてから）の処理」は別のタイミングで実行される。

9-8スタックとキュー — 皿の山とレジの行列

🎯 ゴール：スタック（LIFO）とキュー（FIFO）の違いと、push/pop・enqueue/dequeueの動きを追えるようになる。

スタックは「積んだ皿」のイメージです。後から積んだ皿（後入れ）から先に取る（先出し）のでLIFO（Last In First Out）と呼びます。操作はpush（積む）とpop（取り出す）。

キューは「レジの行列」のイメージです。先に並んだ人（先入れ）から先にレジを通る（先出し）のでFIFO（First In First Out）と呼びます。操作はenqueue（列に並ぶ）とdequeue（列から抜ける）。2025年公開問題の問3ではスタックを配列とポインタで実装する出題実績があります。

✏️ 例題（本試験風） 空のスタックに対して push(A)→push(B)→pop()→push(C)→push(D)→pop()→pop() の順に操作したとき、最終的にスタックに残る要素は何か（底から順に）。
答えを見る
操作スタックの中身（底→top）
push(A)A
push(B)A, B
pop()A　（Bが取り出される）
push(C)A, C
push(D)A, C, D
pop()A, C　（Dが取り出される）
pop()A　（Cが取り出される）

最終的にスタックに残るのはAだけです。

スタックの代表的な応用として、科目Aでも頻出の「逆ポーランド記法（後置記法）」があります。演算子を数の後ろに書く記法で、(A＋B)×C は AB＋C× と表されます。読み方のコツは「数が来たらpush、演算子が来たら2つpopして計算し、結果をpushする」の一文で覚えることです。

✏️ 例題（本試験風） 後置記法（逆ポーランド記法）で書かれた式 3 4 + 2 × の値はいくつか。
答えを見る
読んだ記号操作スタックの中身（底→top）
3push(3)3
4push(4)3, 4
+2つpop（4, 3）して3+4=7を計算しpush7
2push(2)7, 2
×2つpop（2, 7）して7×2=14を計算しpush14

最後にスタックに残った14が式の値です。

⚡ ポイント
  • スタック＝LIFO（後入れ先出し）＝push/pop。9-7の再帰のコールスタックも実はスタックの一種。
  • キュー＝FIFO（先入れ先出し）＝enqueue/dequeue。順番待ちの処理に使われる。
  • 操作列の問題は、1回ずつ表に書き出して中身を更新するのが一番確実。
  • 逆ポーランド記法はスタックの代表応用。『数はpush・演算子は2つpopして計算しpush』の一文で解ける。

9-9連結リスト — 参照のつなぎ替え

🎯 ゴール：連結リストが「値＋次への参照」の集まりであること、挿入/削除が参照のつなぎ替えだけで済むことを理解する。

配列は要素が横一列に並んでいますが、連結リストは各要素（節）が「値」と「次の要素への参照」の2つを持ち、鎖のようにつながっています。リストの終端は公式用語で「未定義の値」と表現されます。2022年サンプル問題の問3では、この連結リストへの要素追加が出題されています。

配列に途中から要素を挿入すると、後ろの要素をすべて1つずつずらす必要がありますが、連結リストなら「次への参照」を書き換えるだけで済みます。これが連結リストの最大の利点です。

⚡ ポイント
  • 連結リストの各要素は「値」＋「次への参照」の組。配列のような連番の添字は持たない。
  • 終端（それ以上つながる要素がない）は「未定義の値」という公式表現で表される。
  • 挿入・削除は参照のつなぎ替えだけで完了し、他の要素を移動させる必要がない。

9-10探索アルゴリズム — 線形探索と二分探索

🎯 ゴール：線形探索と二分探索の違いと、二分探索が探索範囲を毎回半分に絞り込む動きを追えるようになる。

線形探索は先頭から順番に1つずつ調べる方法で、平均するとn個のデータに対しn/2回の比較が必要です。二分探索は、データが整列済みであることを前提に、探索範囲の中央（mid）と比較して範囲を半分ずつ絞り込む方法で、log2 n回程度の比較で済みます（番兵法という探索の効率化テクニックもあります）。

✏️ 例題（本試験風） 整列済みの配列 b ← {2,5,8,11,14,17,20,23,26,29}（1始まり・10個）から二分探索で 29 を探すとき、比較は何回行われるか。中央の添字は (low+high)÷2 の小数点以下切り捨てとする。
答えを見る
比較回数lowhighmidb[mid]判定
111051429>14 → 右へ（low←6）
261082329>23 → 右へ（low←9）
391092629>26 → 右へ（low←10）
410101029一致！発見

比較回数は4回です。10個のデータでも、二分探索なら最大4回程度で見つかることが分かります（log2 10 ≒ 3.3）。

⚡ ポイント
  • 二分探索は「整列済み」が絶対条件。整列されていないデータには使えない。
  • 1回比較するたびに範囲がほぼ半分になる＝計算量はO(log n)（9-12で詳しく）。
  • lowとhighの更新（mid+1／mid-1にするかmidのままにするか）は問題文の定義を必ず確認する。

9-11ソートアルゴリズム — バブル・選択・挿入・クイック・マージ

🎯 ゴール：代表的なソート（並べ替え）アルゴリズムの動き方の違いをイメージできるようになる。

バブルソートは隣り合う要素を比較して、順番が逆なら交換することを繰り返す方法です（比較回数はn(n−1)/2回）。選択ソートは「残りの中から最小（または最大）を選んで並べる」、挿入ソートは「整列済みの部分に1つずつ正しい位置へ挿入する」動きです。クイックソートは基準値（ピボット）より小さいグループと大きいグループに分割して、それぞれの中でも同じ処理を再帰的に繰り返します（2023年公開問題の問3で実装が出題）。マージソートはデータを半分に分割し続け、最後に整列させながら併合していく方法です（2024年公開問題の問4で出題）。

⚡ ポイント
  • バブル・選択・挿入は「素朴だが遅い」グループ（計算量O(n²)）。
  • クイック・マージは「分割して征服する」グループで平均的に速い（計算量O(n log n)）。
  • アルゴリズム名を見て「どういう考え方で並べ替えるか」を思い出せれば、トレースの見通しが立てやすい。

9-12計算量（オーダー） — nが増えたときの重さ

🎯 ゴール：O記法の意味と、データ量nが大きくなったときにどのくらい処理が重くなるかの目安を持てるようになる。

計算量（オーダー、O記法）は「データ量nが増えたときに処理時間がどれくらい増えるか」の目安です。増え方が小さい順に並べると次のようになります。

O(1) ＜ O(log n) ＜ O(n) ＜ O(n log n) ＜ O(n²)

二分探索（9-10）はO(log n)、バブルソート（9-11）はO(n²)、クイックソートは平均するとO(n log n)です。nが小さいうちは差が目立ちませんが、nが大きくなるとO(n²)は急激に重くなります。

⚡ ポイント
  • O記法は「正確な処理時間」ではなく「増え方の目安」。係数や細かい定数は無視して考える。
  • 迷ったら「二分探索=O(log n)、バブル=O(n²)、クイック平均=O(n log n)」の3点セットを軸に思い出す。
  • nが大きい場面（大量データ）ほどO(n²)のアルゴリズムは不利になる。

9-13ハッシュ法 — 計算で置き場所を決める

🎯 ゴール：ハッシュ関数で格納位置を直接計算する仕組みと、衝突が起きたときの2つの解決方法を理解する。

ハッシュ法は、探索するのではなく「計算で格納位置を直接求める」方法です。たとえば h(key) = key mod m（keyをmで割った余り）のような式（ハッシュ関数）を使えば、1回の計算＝O(1)で格納位置が分かります。ただし別のkeyが同じ格納位置を計算してしまう「衝突」が起こることがあり、その解決方法には、空いている別の番地を探すオープンアドレス法と、同じ番地に鎖のようにつなげるチェイン法（数珠つなぎ）の2種類があります。2023年公開問題の問4ではオープンアドレス法による衝突解決が出題されています。

答えを見る

分子＝3×6＋4×8＝18＋32＝50。

9-16木構造と二分木 — 先行順・中間順・後行順

🎯 ゴール：木構造の基本用語と、二分探索木を3つの順番（先行順・中間順・後行順）で走査したときの結果の違いを理解する。

ここからの3項目は、出題範囲に明記されている追加パターンです。9-15までの『型』があれば読めるので、仕上げとして押さえましょう。

木構造とは、データを階層状につないだ構造です。一番上の要素を根（root）、つながっている各要素を節（node）、それ以上子を持たない末端の要素を葉（leaf）と呼び、上下でつながる関係を親子と呼びます。特に子が最大2つまでの木を二分木、さらに「左の子孫＜親＜右の子孫」となるように値を配置した二分木を二分探索木と呼びます。

木をたどる（走査する）方法は3種類あり、名前は「親をいつ訪れるか」で決まります。先行順は親→左→右、中間順は左→親→右、後行順は左→右→親の順にたどります。この中で最も重要なのが「二分探索木を中間順で走査すると、値が必ず昇順に並ぶ」という性質で、科目Bで最頻出のポイントです。

関連してヒープという構造も知っておきましょう。ヒープは「親が常に子より大きい（または常に子より小さい）」というルールを持つ完全二分木で、最大値（または最小値）が必ず根にあるため、一番大きい（小さい）値をすぐに取り出せます。

✏️ 例題（本試験風） 上の図の二分探索木を中間順で走査したとき、出力される値の順番はどれか。
答えを見る

答えは 1 → 3 → 6 → 8 → 10 → 14です。中間順（左→親→右）は「左の子孫を全部訪れてから親、その後右の子孫」という順なので、二分探索木の性質どおり必ず昇順になります。

参考までに、先行順（親→左→右）でたどると8, 3, 1, 6, 10, 14、後行順（左→右→親）でたどると1, 6, 3, 14, 10, 8になります。同じ木でも訪れる順番が変わると出力順が変わることを確認しておきましょう。

⚡ ポイント
  • 走査の名前は「親をいつ訪れるか」で決まる：先行順＝先に親、中間順＝真ん中で親、後行順＝後で親。
  • 「二分探索木を中間順で走査すると昇順になる」は即答できるレベルで覚えておく。
  • 後行順（左→右→親）は、式を木の形で表した「式の木」を9-8の逆ポーランド記法へ変換するときのたどり方でもある。

9-17グラフと隣接行列 — つながりを表として持つ

🎯 ゴール：グラフの基本用語（節点・辺・無向/有向）を知り、隣接行列への変換ができるようになる。

グラフとは、いくつかの点（節点・ノード）を線（辺・エッジ）でつないだ構造です。路線図や友達関係のネットワークなどが身近な例です。辺に向きがないものを無向グラフ、向きがあるものを有向グラフと呼びます。

プログラムでグラフを扱うときの定番は、二次元配列で表す隣接行列です。節点iと節点jの間に辺があればm[i,j] ＝ 1、なければ0とします。無向グラフの場合、iからjへの辺とjからiへの辺は同じ1本なのでm[i,j] ＝ m[j,i]となり、行列は対角線に対して対称になります。2024年公開問題 問3では「辺の一覧から隣接行列へ変換する」処理が出題されています。

✏️ 例題（本試験風・穴埋め型） 次の手続きは、辺の一覧（edge配列）から隣接行列mを作成するものである。edge[i,1]とedge[i,2]はi本目の辺の両端の節点番号、mはすべて0で初期化されたn×nの二次元配列とする。［　a　］・［　b　］に入る字句の適切な組合せはどれか。 
/* edge[i,1] と edge[i,2] は i本目の辺の両端の節点番号 */
/* m はすべて0で初期化された n×n の二次元配列 */
整数型: i
for (i を 1 から 辺の本数 まで 1 ずつ増やす)
  m[edge[i,1], edge[i,2]] ← 1
  m[［ a ］, ［ b ］] ← 1   /* 無向グラフなので逆向きも1にする */
endfor
ab
アedge[i,1]edge[i,2]
イedge[i,2]edge[i,1]
ウedge[i,1]edge[i,1]
エedge[i,2]edge[i,2]
答えを見る

答えはイです。

1行目のm[edge[i,1], edge[i,2]] ← 1で片方向（例：辺{2,3}ならm[2,3]）を1にしているので、2行目のコメントにある「逆向きも1にする」を実現するには、行と列を入れ替えたm[edge[i,2], edge[i,1]]（イ）が必要です。

アは1行目とまったく同じマスをもう一度1にするだけで、逆向きのマスが更新されません。ウ・エはedge[i,1], edge[i,1]やedge[i,2], edge[i,2]のように行と列が同じ節点番号になり、対角線上（自分自身への辺）を立ててしまう誤りです。

解き方のコツは、まず選択肢の差分（どの添字がedge[i,1]かedge[i,2]か）だけを比較し、具体的な辺（例：節点2と3を結ぶ辺）を1つ当てはめて、どの選択肢が「m[2,3]とm[3,2]の両方を1にする」動きになるかを消去法で絞り込むことです。

⚡ ポイント
  • 無向グラフの隣接行列は対角線に対して対称（m[i,j]＝m[j,i]）になる。
  • 有向グラフの場合は「行＝辺の出発点、列＝辺の到着点」で、対称にはならない。
  • 見た目は新しくても、正体は9-5で学んだ二次元配列の読み書きそのもの。
