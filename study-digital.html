💡 デジタル回路

デジタルの世界は「0か1か」の2択だけ。論理ゲートで判断し、フリップフロップで記憶する——コンピュータの中身も、この2つの組み合わせでできています。

3-1論理式 — AND・OR・NOTを式で書く

🎯 ゴール：基本ゲートの記号・論理式・動作をセットで言える

デジタル回路の判断はすべてAND（〜かつ〜）・OR（〜または〜）・NOT（〜でない）の組み合わせ。論理式はそれを数式の形で書いたものです。ANDは「・」、ORは「＋」、NOTは上線（バー）で表します。

⚡ ポイント
  • 書き方：AND＝A・B、OR＝A＋B、NOT＝上線（バー）。例：Y＝A・B＋C。
  • 記号の見分け：ANDは平ら＋丸い頭、ORは反った入口、○付きは否定。
  • NAND・NORは「AND・ORの出力をNOTしたもの」。NANDだけであらゆる論理が作れる。

3-2真理値表 — 全パターンの一覧表

🎯 ゴール：入力の全組み合わせを漏れなく書き出し、出力を埋められる

真理値表は「入力の全組み合わせ」に対する出力を一覧にした表。入力がn個なら行数は2ⁿ行（2入力＝4行、3入力＝8行）。組み合わせは2進数のカウントアップ順（00→01→10→11）で書くと絶対に漏れません。

✏️ 例：Y＝A・B＋C（AかつB、またはC）の真理値表（3入力＝8行） まず自分で書いてみよう。行は2進数の順（000〜111）で8行。途中の列（A・B）を先に作ってから最終出力Yを埋めるとミスが減る。
答えを見る
ABCA・BY＝A・B＋C
00000
00101
01000
01101
10000
10101
11011
11111
検算：Yが1になるのは「Cが1」の4行＋「AもBも1でCが0」の1行（110）＝計5行 ✓
⚡ ポイント
  • 行数＝2ⁿ。2進数の順に書くのが鉄則（000,001,010,011,100…）。
  • 「回路 → 真理値表 → 論理式」も「論理式 → 真理値表 → 回路」も、どちら向きにも行き来できるのがゴール。
  • 真理値表が一致すれば、見た目の違う回路でも「同じ働き」と言える。

3-3各種ディジタルIC（74シリーズ） — ゲートを「部品」として買う

🎯 ゴール：代表的な74シリーズICの中身とピンの数え方がわかる

論理ゲートは「74シリーズ」というICファミリで部品化されています。1つのパッケージにゲートが複数入っているのが普通です。

型番中身覚え方
74002入力NAND × 4個いちばん基本。「00＝NAND」
7404NOT（インバータ）× 6個ひっくり返す6兄弟
74082入力AND × 4個—
74322入力OR × 4個—
74862入力XOR × 4個一致・不一致の判定に
⚡ ポイント
  • ピン番号は目印（切り欠き・ドット）を上にして左上が1番、反時計回り。14ピンなら右上が14番。
  • 多くの14ピンロジックICはGND＝7番、VCC＝14番（例外もあるので必ずデータシート確認）。
  • 74LS…（TTL）、74HC…（CMOS）などの中間文字は「作り」の違い。同じ番号なら論理機能は同じ。

3-4ディジタルICを使った回路作成 — 電源・入力・出力の「お作法」

🎯 ゴール：スイッチ入力→ゲート→LED出力の回路を組める

「真理値表 → 論理式 → ゲートのつなぎ」と決めたら、実際の配線ではロジック以外の3点セット——入力の処理・出力の電流制限・電源まわり——が成功のカギになります。

⚡ ポイント
  • 入力を開放（宙ぶらりん）にしない：スイッチにはプルダウン（またはプルアップ）抵抗。未使用入力もHかLに固定する。
  • LEDには電流制限抵抗（330Ωなど）を必ず直列に。
  • パスコン（0.1µF）をICのVCC-GND間近くに。動作を安定させるお守り。
  • 組む順序：電源配線 → 動作確認（VCC-GND電圧）→ 入力回路 → ロジック → 出力回路。

3-5正論理 — Hを1とみなす

🎯 ゴール：電圧のH/Lと論理の1/0の対応を説明できる

デジタル信号の実体は電圧です。電圧が高い状態をH（High）、低い状態をL（Low）と呼び、H＝1、L＝0と割り当てるのが正論理。ふだんの論理回路はほぼ正論理で考えます。

⚡ ポイント
  • H/Lは電圧の話、1/0は意味の話。この2階建てを分けて考えるのがコツ。
  • HとLの間には「不定」の帯がある。中途半端な電圧は0でも1でもない（だから入力を開放にしない）。

3-6負論理 — Lを1とみなす（アクティブ・ロー）

🎯 ゴール：バー付き信号・○付きピンの意味を読み取れる

負論理はL＝1（有効）、H＝0とみなす約束。「Lになったときに動く」信号はアクティブ・ローと呼ばれ、リセットやチップ選択など制御信号の定番です。

見た目意味
信号名の上線：RESET、CSLにするとリセット／選択が働く（アクティブ・ロー）
ICピンの小さな○そのピンは負論理（Lで有効・出力ならLを出す）
○も上線もない正論理（Hで有効）
✏️ 読み取り練習 RESET端子付きのカウンタIC。ふだんこの端子はH・Lどちらにしておく？
答えを見る Hにしておく（＝リセット無効で普通に動作）。Lに落とした瞬間リセットがかかる。バー付き＝「Lで発動」と読む。
⚡ ポイント
  • 負論理は「壊れて信号線がGNDに落ちたときに安全側に倒れる」「ノイズに強い」などの理由で制御信号に多い。
  • 回路図を読むとき、バーと○を見つけたら「Lで有効」と翻訳すれば混乱しない。

3-72進数 — 0と1だけの数

🎯 ゴール：位の重みを使って2進数を読める

デジタル回路はH/Lの2状態しか持てないので、数は0と1だけで表す2進数を使います。各桁には重み（1, 2, 4, 8, 16…と倍々）があり、「1が立っている桁の重みを足す」と10進数になります。

⚡ ポイント
  • 重みは右から 1, 2, 4, 8, 16, 32, 64, 128。この8つはそのまま暗記してよい。
  • nビットで表せるのは0〜2ⁿ−1（4ビット＝0〜15、8ビット＝0〜255）。
  • 2進数の1桁＝ビット。デジタル回路の信号線1本が1ビットに対応する。

3-816進数 — 4ビットを1文字で

🎯 ゴール：0〜F と 4ビットの対応がわかる

2進数は桁が長くて読みにくいので、4ビットをまとめて1桁で表す16進数をよく使います。使う記号は0〜9とA〜Fの16種類（A=10, B=11, C=12, D=13, E=14, F=15）。

10進01234567
2進00000001001000110100010101100111
16進01234567
10進89101112131415
2進10001001101010111100110111101111
16進89ABCDEF
⚡ ポイント
  • 16進1桁＝2進4桁（きっちり対応）。だからプログラムやデータシートで多用される。
  • 書き方の目印：0x3F・3FH などは16進の意味。
  • まず A=1010、C=1100、F=1111 あたりを覚えると速くなる。

3-9進数変換 — 3つの方法で行き来する

🎯 ゴール：2進⇄10進⇄16進を自力で変換できる

2進・16進はデジタル回路の住所表記のようなもので、データシートを読むときもカウンタの動作を確認するときも、3つの進数を行き来しながら読むことになります。覚える変換は「重みを足す」「2で割る」「4ビット区切り」の3パターンだけです。

✏️ 例題 ① 2進 11111111 を10進と16進に　② 16進 A3 を2進に
答えを見る ① 128+64+32+16+8+4+2+1＝255。4ビットずつ区切ると 1111|1111 → FF。
② A＝1010、3＝0011 → 10100011。10進なら 163。
⚡ ポイント
  • 10進→16進は「一度2進を経由する」か「16で割って余り」。2進経由が確実。
  • 区切りは必ず右（下位）から4ビットずつ。足りない分は左に0を補う。

3-107セグメントLED — 棒7本で数字を描く表示器

🎯 ゴール：セグメント名（a〜g）と点灯パターン、コモンの種類がわかる

7セグメントLEDは7本の棒状LED（＋小数点dp）の点灯パターンで数字を表す表示器。中身はただのLEDが8個入っているだけ、と考えると怖くありません。

⚡ ポイント
  • 「8」は全セグメント、「1」はbとcだけ。0〜9のパターンは自分で一度書き出すと一気に定着する。
  • カソードコモン＝−が共通（Hで点灯）／アノードコモン＝＋が共通（Lで点灯）。買う前・使う前に必ず確認。
  • dp（小数点）を含めると中身はLED8個。1本ずつはただのLEDなので極性と電流制限が必要。

3-117segLEDを使った回路作成 — 点灯パターンはデコーダICに丸投げ

🎯 ゴール：デコーダICを使って数値を表示する構成を説明できる

7本のセグメントを1本ずつ自力で制御するのは大変。そこで「BCD（2進化10進）の4ビットを入れると、a〜gの7本を自動で作ってくれる」デコーダICを使います。

⚡ ポイント
  • 4511はカソードコモン用（出力H）、74LS47などはアノードコモン用（出力L）。7segの種類とペアで選ぶ。
  • 電流制限抵抗はセグメント1本ごとに入れる（1本にまとめると点灯本数で明るさが変わってしまう）。
  • カウンタIC（後述）→デコーダ→7segとつなぐと「押した回数を数えて表示する」回路が完成する。

3-12フリップフロップ — 1ビットを記憶する

🎯 ゴール：RS・D・JKの違いと動作表がわかる

ゲートは「今の入力」だけで出力が決まりますが、フリップフロップ（FF）は1ビット（0か1）を覚えておける記憶回路。カウンタもレジスタもメモリも、元をたどればFFの集まりです。

⚡ ポイント
  • RS-FF：セット/リセットの基本形。S＝R＝1は禁止入力。
  • D-FF：クロックの立ち上がりエッジでDをQへ。「1ビットのカメラのシャッター」。
  • JK-FF：禁止がなく、J＝K＝1でトグル（反転）。この反転がカウンタの心臓部になる。

3-13フリップフロップで作る回路 — カウンタ

🎯 ゴール：T-FFをつなぐと2進カウンタになる理由を波形で説明できる

JK-FFをJ＝K＝1（T-FF）にすると「クロックのたびに反転」します。反転＝周波数が半分（2分周）。これを数珠つなぎにすると、各段が1/2, 1/4, 1/8…となり、そのまま2進数のカウントアップになります。

⚠️ 実験でハマりがち：チャタリング 押しボタンをクロックにすると、接点が跳ねて1回押しただけで数回カウントが進むことがある（チャタリング）。コンデンサやシュミットトリガICで対策するか、「そういう現象がある」と知っておくだけでも切り分けが速くなる。
⚡ ポイント
  • FFをn個つなぐとnビットカウンタ（0〜2ⁿ−1）。3個で0〜7、4個で0〜15。
  • 1段ごとに周波数1/2 → カウンタは分周器としても使われる（時計の1秒作りなど）。
  • D-FFのQ̄をDに戻してもT-FF（トグル）が作れる。

3-14カウンタIC — クロックのたびに＋1してくれる部品

🎯 ゴール：カウンタICの役割と代表的な端子がわかる

FFを自分でつながなくても、1チップでカウンタになっているICがあります。クロックが入るたびに2進数で＋1（または−1）してくれる部品です。

型番（例）種類特徴
74HC1614ビット2進（16進）カウンタ0〜15を数える。同期式・プリセット可
749010進カウンタ0〜9で一周 → 7seg表示と相性がよい
74HC193アップダウンカウンタ＋1も−1もできる
⚡ ポイント
  • 主な端子：CLK（クロック入力）・Q0〜Q3（2進出力）・CLR（リセット：0に戻す）・LD（プリセット：好きな値から開始）。バー付き＝アクティブ・ロー（Lで動作）が多い。
  • カウンタIC → デコーダIC → 7seg とつなぐと「数えて表示」が完成。前項の回路とつながる。
  • 10進カウンタは「9の次に0へ戻る」ようにFF＋リセットが内蔵されている、と考えればよい。

3-15レジスタ — 複数ビットをまとめて保持

🎯 ゴール：シフトレジスタの動きをクロックごとに追える

D-FFを並べて複数ビットをまとめて記憶する回路がレジスタ。中でも、クロックごとにデータを隣へ送っていくシフトレジスタは「バケツリレーの記憶装置」です。

✏️ 動きを追ってみる（1→0→1→1 の順に入力） はじめ0000の4ビットシフトレジスタに、1→0→1→1 の順で1クロックずつ入力する。各クロック後の Q0〜Q3 を自分で追ってから答え合わせしよう。ヒント：クロックのたびに全FFが一斉に「左隣の値」を取り込む。
答えを見る
クロック入力Q0Q1Q2Q3
はじめ—0000
1回目11000
2回目00100
3回目11010
4回目11101
データが1クロックずつ右へスライドしていくのがわかる。
⚡ ポイント
  • レジスタ＝D-FFの集まり。並列に一斉保存するタイプと、1ビットずつ送るシフトタイプがある。
  • シフトレジスタIC（74HC164、74HC595など）は「線3本で LED をたくさん制御」など、少ないピンでデータを送るのに大活躍。
  • カウンタは「数を覚えるFF」、レジスタは「データを覚えるFF」。どちらもFFの応用と捉えると整理できる。

Q1. AND・OR・NOTの論理式の書き方は？
AND＝A・B、OR＝A＋B、NOT＝A（上線）。 → 解説 3-1 に戻る
Q2. 2入力の真理値表は何行？3入力では？
2ⁿ行なので、2入力＝4行、3入力＝8行。2進数のカウント順（00,01,10,11）に書くと漏れない。 → 解説 3-2 に戻る
Q3. NANDゲートの出力がLになるのはどんなとき？
入力が両方H（1・1）のときだけ。それ以外はすべてH。ANDの逆と覚える。 → 解説 3-1 に戻る
Q4. 7400はどんなIC？VCCとGNDは何番ピン？
2入力NANDが4個入ったIC。VCC＝14番、GND＝7番（切り欠きを上にして左上が1番、反時計回り）。 → 解説 3-3 に戻る
Q5. 使わない入力ピンはどうする？
HかLに固定する（開放にしない）。宙ぶらりんだとレベルが不定になり誤動作の原因になる。 → 解説 3-4 に戻る
Q6. 正論理と負論理の違いは？
正論理＝H を1とする。負論理＝L を1とする。電圧（H/L）と意味（1/0）の割り当て方の違い。 → 解説 3-5 に戻る
Q7. RESET（バー付き）端子の意味は？
Lにするとリセットがかかる（アクティブ・ロー）。ふだんはHにしておく。 → 解説 3-6 に戻る
Q8. 2進数 1101 を10進数に。
8＋4＋1＝13（16進ならD）。 → 解説 3-7 に戻る
Q9. 10進数 25 を2進数に。
2で割って余りを下から読む → 11001（16＋8＋1で検算OK）。 → 解説 3-9 に戻る
Q10. 2進数 11111111 を16進数と10進数に。
1111|1111 → FF。10進では255（8ビットの最大値）。 → 解説 3-9 に戻る
Q11. 7segで「1」を出すとき点灯するセグメントは？
bとc（右側の縦2本）。ちなみに「8」は7本全部。 → 解説 3-10 に戻る
Q12. アノードコモンとカソードコモンの違いは？
アノードコモン＝＋側が共通（セグメントをLにすると点灯）、カソードコモン＝−側が共通（Hで点灯）。デコーダICもそれに合わせて選ぶ（4511はカソードコモン用）。 → 解説 3-10 に戻る
Q13. D-FFがDを取り込むのはいつ？
クロックの立ち上がりエッジの瞬間。それ以外ではDが変化してもQは保持される。 → 解説 3-12 に戻る
Q14. JK-FFでJ＝K＝1にすると？
クロックのたびに出力が反転（トグル）する。これがT-FFで、カウンタ（2分周）の基本になる。 → 解説 3-12 に戻る
Q15. FFを3個つないだカウンタは何がどこまで数えられる？
3ビット2進カウンタで0〜7（＝8分周）。n個なら0〜2ⁿ−1。 → 解説 3-13 に戻る
Q16. シフトレジスタはどんな動きをする？
クロックごとにデータを隣のFFへ1つずつ送って保持する。直列で入れたデータを並列で取り出す、などピン数の節約に使われる。 → 解説 3-15 に戻る
